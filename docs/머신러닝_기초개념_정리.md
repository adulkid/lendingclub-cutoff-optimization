# 머신러닝 기초 개념 정리 (PD 모델링용)

## 전체 흐름

```
[1] 지도학습 기본 틀      [2] 알고리즘 두 종류        [3] 학습/검증 원칙        [4] 평가지표
X(66 feature) → y(target)  로지스틱회귀 vs 트리기반    train/test, 과적합 방지    AUC·KS·Calibration
                                                              ↓
                                                    우리 프로젝트: 2007~2014 train / 2015 test
```

## 1. 지도학습(Supervised Learning) 기본 틀

- **정의**: 정답(y)이 붙어있는 데이터로, 입력(X)과 정답(y) 사이의 관계를 학습하는 방식. 학습 후엔 정답을 모르는 새 데이터의 y를 예측.
- **우리 프로젝트 대입**: X = feature 66개(dti, fico, avg_cur_bal 등), y = target(부도=1/정상=0). 모델은 "이 사람 정보(X)를 보면 부도확률(y=1일 확률)이 얼마냐"를 학습.
- 참고(따로 학습): 비지도학습(군집화 등)은 우리 프로젝트에선 안 씀 — 정답(target)이 이미 있는 분류 문제라서.

## 2. 로지스틱 회귀 (Logistic Regression)

- **정의**: 입력들의 가중합(선형결합)을 0~1 확률로 눌러주는 함수(시그모이드)를 씌운 모델. 출력 자체가 "확률"로 해석 가능.
- **왜 중요한가**: 계수(coefficient) 하나하나가 "이 feature가 1 증가하면 부도 오즈(odds)가 몇 배 되는가"로 해석 가능 — 여신심사팀에 "왜 이 결정이 나왔는지" 설명하기 쉬움. 확률 출력이 원래 부드럽고 안정적이라 calibration도 상대적으로 잘 됨.
- **우리 프로젝트 대입**: Model A/B의 "로지스틱" 축. 지난번 정한 hinge(avg_cur_bal), 이진더미(mort_acc), sentinel+flag(결측치 규칙)가 전부 이 모델에만 해당하는 전처리 — 로지스틱은 "선형" 가정이라 비선형 패턴을 우리가 직접 변수로 만들어 넣어줘야 하기 때문.
- 참고(따로 학습): L1/L2 정규화(regularization) — 계수가 너무 커지는 걸 억제해서 과적합 방지하는 옵션. sklearn `LogisticRegression`에 기본 내장돼 있음.

## 3. 트리 기반 모델 (Decision Tree → LightGBM 등)

- **정의**: "이 feature가 X 이상이면 왼쪽, 미만이면 오른쪽" 식으로 데이터를 계속 쪼개나가는 규칙 기반 모델. LightGBM/XGBoost는 이런 트리 수백~수천 개를 순차적으로 더해가며 오차를 줄이는 방식(gradient boosting).
- **왜 중요한가**: 트리는 어떤 지점에서든 자유롭게 쪼갤 수 있어서, **비선형·문턱효과·비단조 패턴을 원값 그대로도 자동으로 잡아냄**. 결측치도 특정 방향으로 자동 분기 처리 가능.
- **우리 프로젝트 대입**: 이게 바로 지난번 avg_cur_bal(역U자)·mort_acc(문턱효과) 구간화 논의에서 "트리는 상관없다"고 한 이유. 로지스틱은 우리가 hinge/더미로 직접 비선형을 만들어줘야 했지만, 트리 버전 Model A/B는 원값 그대로 넣어도 됨.
- 참고(따로 학습): feature importance 두 방식(split 횟수 기준 vs gain/정보이득 기준) — 결과 해석 시 어느 걸 볼지는 나중에 실제 모델 돌릴 때 다시 짚어줄게. SHAP value(개별 예측 기여도 분해)도 심화 주제라 필요할 때 별도 설명.

## 4. Train/Test 분할과 과적합

- **정의**: 모델이 학습 데이터의 패턴을 "일반적 규칙"이 아니라 "그 데이터에만 있는 우연한 잡음"까지 외워버리는 현상을 과적합(overfitting)이라 함. 이걸 감지하려고 학습에 안 쓴 데이터(test)로 따로 검증.
- **왜 중요한가**: train에서 성능이 아무리 좋아도 test에서 뚝 떨어지면 실전(미래 데이터)에서도 못 씀 — 신용평가는 미래 대출 심사에 쓰는 모델이라 이 검증이 특히 중요.
- **우리 프로젝트 대입**: 이미 확정한 **2007~2014 train(2007~2011은 early_era 플래그) / 2015 test(주) / 2016(보조)**가 바로 이 원칙의 구현. 시간 순으로 자른 이유는 랜덤 분할 시 미래 정보가 새어 들어가기 때문(이전에 논의한 대로).
- 참고(따로 학습): cross-validation(교차검증) — train을 여러 겹으로 나눠 반복 검증하는 기법. 우리는 시계열 특성상 단순 cross-validation보다 out-of-time 분할이 우선이라, 지금 단계에선 필수 아님.

## 5. 평가지표

- **AUC (Area Under ROC Curve)**: 무작위로 부도 1명·정상 1명을 뽑았을 때, 모델이 부도자에게 더 높은 점수를 줄 확률. 0.5=무작위, 1.0=완벽. **"순위를 얼마나 잘 매기는가"**를 보는 지표.
- **KS(Kolmogorov-Smirnov) 통계량**: 부도자와 정상자의 점수 분포가 얼마나 잘 갈라지는지 최대 격차로 측정. 신용평가업계에서 AUC와 함께 표준적으로 쓰는 지표.
- **Calibration(보정)**: 이미 논의했던 개념 — 예측확률이 실제 발생비율과 일치하는지. AUC/KS와 달리 "확률 절댓값의 정확도"를 봄.
- **우리 프로젝트 대입**: AUC·KS는 Model A/B 4개 조합의 순위 예측력 비교용. **calibration은 그중에서도 제일 중요** — 최종적으로 뽑힌 모델의 PD를 EL=PD×LGD×EAD에 절댓값 그대로 넣을 거라서.
- 참고(따로 학습): Precision/Recall/F1 — 분류 임계값을 하나 정해놓고 보는 지표들인데, 우리는 "cutoff 자체가 분석의 결과물"이라 이런 고정 임계값 지표보다 AUC/KS/calibration이 더 적합. 필요하면 나중에 참고.

## 6. Model A/B 실험과의 연결 정리

| 축 | Model A (grade 포함) | Model B (grade 제외) |
|---|---|---|
| 로지스틱 | hinge/더미/sentinel 전처리 적용, grade 계수 해석 가능 | 동일 전처리, grade 없이 raw feature들이 얼마나 설명하는지 확인 |
| 트리(LightGBM) | 원값 그대로, grade importance 확인 | 동일, grade 없이도 성능 유지되는지 확인 |

4개 조합 전부 학습 후 AUC·KS로 순위력 비교, calibration으로 확률 신뢰도 확인 → 최종 모델 선정.
